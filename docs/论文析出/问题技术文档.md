# 基于《毕业设计AI稿》析出的项目问题技术文档

## 1. 文档目的

本文档用于从 `毕业设计AI稿.md` 中提炼论文明确提到的“问题、痛点、缺陷、限制、待改进项”，并按技术分析方式整理为可追溯的说明文档。

本文档只依据论文文本本身进行归纳，不额外引入未在论文中明确表述的新结论。

## 2. 提取范围与判定规则

- 提取范围：摘要、绪论、需求获取、系统分析、系统设计、系统实现、系统测试、结论与展望中与“问题”直接相关的表述。
- 计入标准：论文中明确描述为痛点、短板、限制、缺陷、局限、改进方向的内容。
- 不计入内容：纯背景介绍、无问题含义的功能说明、未明确指出为问题的中性设计描述。
- 说明原则：同义或重复表述进行合并，避免把同一问题拆成多个近似条目。

## 4. 问题归并后的技术主线

从论文整体叙述来看，所有问题可以归并为 5 条技术主线：

1. 编辑体验主线：解决编辑与预览割裂、多模式一致性与高频交互效率问题。
2. 文件管理主线：补足资源管理器级目录浏览、定位、恢复与文件组织能力。
3. 跨端协同主线：处理增量同步、版本冲突、断网恢复与设备绑定问题。
4. 跨平台适配主线：统一桌面路径语义与 Android SAF 文档语义。
5. 工程质量主线：优化大文件性能、自动化测试、安全策略与产品化能力。

## 5. 使用建议

- 若用于答辩 PPT，可优先选取 `P-01` 至 `P-09` 作为“研究背景与痛点”。
- 若用于项目改造清单，可重点参考 `P-16` 至 `P-41` 作为“当前不足与后续路线”。
- 若要继续落为研发任务，可把每个问题编号直接映射到后续的需求或 issue。
