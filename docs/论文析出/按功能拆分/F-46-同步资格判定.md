# 同步资格判定

## 功能是什么

仅对已具备云端标识或被加入书签的文件启用同步。

## 对应代码在哪

- 代码位置：[syncEngine.js](../../../src/services/syncEngine.js)
  符号：`registerLocalDocument`
  作用：只有收藏文件或已有 fileId 的文件才正式进入云身份体系。
- 代码位置：[syncEngine.js](../../../src/services/syncEngine.js)
  符号：`queueLocalUpsert`
  作用：同步前检查当前文件是否满足同步资格。
- 代码位置：[useFileIdStore.js](../../../src/store/useFileIdStore.js)
  符号：`getOrCreate`
  作用：为满足同步资格的本地路径分配稳定 fileId。

## 关键代码怎么读

- 第一步看 [syncEngine.js](../../../src/services/syncEngine.js) 中的 `registerLocalDocument`，其作用是：只有收藏文件或已有 fileId 的文件才正式进入云身份体系。
- 第二步看 [syncEngine.js](../../../src/services/syncEngine.js) 中的 `queueLocalUpsert`，其作用是：同步前检查当前文件是否满足同步资格。
- 第三步看 [useFileIdStore.js](../../../src/store/useFileIdStore.js) 中的 `getOrCreate`，其作用是：为满足同步资格的本地路径分配稳定 fileId。
