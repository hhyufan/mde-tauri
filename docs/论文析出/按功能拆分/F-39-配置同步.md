# 配置同步

## 功能是什么

同步主题、字体、编辑器参数等配置项。

## 对应代码在哪

- 代码位置：[syncEngine.js](../../../src/services/syncEngine.js)
  符号：`setupConfigSubscriptions / syncConfig`
  作用：监听本地设置变更并做双向同步。
- 代码位置：[settingsSync.js](../../../src/utils/settingsSync.js)
  符号：`getLocalSettingsSnapshot / applySettingsSnapshot`
  作用：序列化和回放设置快照。
- 代码位置：[sync.controller.ts](../../../mde-server/src/sync/sync.controller.ts)
  符号：`getConfig / updateConfig`
  作用：服务端配置同步接口。
- 代码位置：[sync-config.schema.ts](../../../mde-server/src/schemas/sync-config.schema.ts)
  符号：`SyncConfig`
  作用：配置同步的数据库模型。

## 关键代码怎么读

- 第一步看 [syncEngine.js](../../../src/services/syncEngine.js) 中的 `setupConfigSubscriptions / syncConfig`，其作用是：监听本地设置变更并做双向同步。
- 第二步看 [settingsSync.js](../../../src/utils/settingsSync.js) 中的 `getLocalSettingsSnapshot / applySettingsSnapshot`，其作用是：序列化和回放设置快照。
- 第三步看 [sync.controller.ts](../../../mde-server/src/sync/sync.controller.ts) 中的 `getConfig / updateConfig`，其作用是：服务端配置同步接口。
- 第四步看 [sync-config.schema.ts](../../../mde-server/src/schemas/sync-config.schema.ts) 中的 `SyncConfig`，其作用是：配置同步的数据库模型。
