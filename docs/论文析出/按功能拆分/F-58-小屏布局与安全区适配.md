# 小屏布局与安全区适配

## 功能是什么

针对手机端处理全屏分段布局、安全区和可视区域变化。

## 对应代码在哪

- 代码位置：[useResponsiveLayout.js](../../../src/hooks/useResponsiveLayout.js)
  符号：`useResponsiveLayout`
  作用：负责小屏布局判定。
- 代码位置：[useViewportInsets.js](../../../src/hooks/useViewportInsets.js)
  符号：`useViewportInsets`
  作用：负责安全区和软键盘 inset。
- 代码位置：[App.scss](../../../src/assets/styles/App.scss)
  符号：`mobile layout rules`
  作用：负责根布局在移动端的高度、底部 inset 和侧栏样式。
- 代码位置：[SettingsModal.jsx](../../../src/components/overlays/SettingsModal.jsx)
  符号：`fullScreen mode`
  作用：小屏下把设置页切换成全屏布局。
- 代码位置：[search-modal.scss](../../../src/components/overlays/search-modal.scss)
  符号：`search fullscreen`
  作用：小屏下搜索改为全屏结果页。
- 代码位置：[MainActivity.kt](../../../src-tauri/gen/android/app/src/main/java/com/mde/app/MainActivity.kt)
  符号：`enableEdgeToEdge`
  作用：为前端安全区样式提供原生前提。

## 关键代码怎么读

- 第一步看 [useResponsiveLayout.js](../../../src/hooks/useResponsiveLayout.js) 中的 `useResponsiveLayout`，其作用是：负责小屏布局判定。
- 第二步看 [useViewportInsets.js](../../../src/hooks/useViewportInsets.js) 中的 `useViewportInsets`，其作用是：负责安全区和软键盘 inset。
- 第三步看 [App.scss](../../../src/assets/styles/App.scss) 中的 `mobile layout rules`，其作用是：负责根布局在移动端的高度、底部 inset 和侧栏样式。
- 第四步看 [SettingsModal.jsx](../../../src/components/overlays/SettingsModal.jsx) 中的 `fullScreen mode`，其作用是：小屏下把设置页切换成全屏布局。
- 最后回看其余支撑文件，补齐该功能的边界条件、状态变化和平台差异。
