# 云同步模块

## 功能是什么

负责身份认证、文档同步、配置同步和冲突处理。

## 对应代码在哪

- 代码位置：[syncEngine.js](../../../src/services/syncEngine.js)
  符号：`SyncEngine`
  作用：负责前端同步调度、冲突处理、配置同步和状态切换。
- 代码位置：[useSyncStore.js](../../../src/store/useSyncStore.js)
  符号：`useSyncStore`
  作用：负责同步文档镜像、mutation 队列、冲突和错误状态。
- 代码位置：[useAuthStore.js](../../../src/store/useAuthStore.js)
  符号：`useAuthStore`
  作用：负责登录态、令牌刷新和认证信息持久化。
- 代码位置：[sync.controller.ts](../../../mde-server/src/sync/sync.controller.ts)
  符号：`SyncController`
  作用：负责文档同步与配置同步的 HTTP 接口。
- 代码位置：[sync.service.ts](../../../mde-server/src/sync/sync.service.ts)
  符号：`SyncService`
  作用：负责服务端版本控制、幂等、冲突和设备绑定逻辑。
- 代码位置：[auth.service.ts](../../../mde-server/src/auth/auth.service.ts)
  符号：`AuthService`
  作用：负责认证、密码校验和 JWT 签发。

- 第一步看 [syncEngine.js](../../../src/services/syncEngine.js) 中的 `SyncEngine`，其作用是：负责前端同步调度、冲突处理、配置同步和状态切换。
- 第二步看 [useSyncStore.js](../../../src/store/useSyncStore.js) 中的 `useSyncStore`，其作用是：负责同步文档镜像、mutation 队列、冲突和错误状态。
- 第三步看 [useAuthStore.js](../../../src/store/useAuthStore.js) 中的 `useAuthStore`，其作用是：负责登录态、令牌刷新和认证信息持久化。
- 第四步看 [sync.controller.ts](../../../mde-server/src/sync/sync.controller.ts) 中的 `SyncController`，其作用是：负责文档同步与配置同步的 HTTP 接口。
- 最后回看其余支撑文件，补齐该功能的边界条件、状态变化和平台差异。
