# 基于版本号的冲突检测

## 功能是什么

使用 `rev/baseRev` 比较识别多端并发修改产生的冲突。

## 对应代码在哪

- 代码位置：[syncEngine.js](../../../src/services/syncEngine.js)
  符号：`getRemoteConflictDecision / applyRemoteDoc`
  作用：前端检测本地正文和远端正文是否冲突。
- 代码位置：[syncEngine.js](../../../src/services/syncEngine.js)
  符号：`processMutation 409 branch`
  作用：服务端返回 revision conflict 时在前端生成冲突对象。
- 代码位置：[sync.service.ts](../../../mde-server/src/sync/sync.service.ts)
  符号：`throwRevisionConflict / pushFile rev check`
  作用：服务端基于 baseRev 和当前 rev 检测冲突。

## 关键代码怎么读

- 第一步看 [syncEngine.js](../../../src/services/syncEngine.js) 中的 `getRemoteConflictDecision / applyRemoteDoc`，其作用是：前端检测本地正文和远端正文是否冲突。
- 第二步看 [syncEngine.js](../../../src/services/syncEngine.js) 中的 `processMutation 409 branch`，其作用是：服务端返回 revision conflict 时在前端生成冲突对象。
- 第三步看 [sync.service.ts](../../../mde-server/src/sync/sync.service.ts) 中的 `throwRevisionConflict / pushFile rev check`，其作用是：服务端基于 baseRev 和当前 rev 检测冲突。
