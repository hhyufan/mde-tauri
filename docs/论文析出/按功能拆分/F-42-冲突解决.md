# 冲突解决

## 功能是什么

用户可选择保留本地版本或采用远程版本恢复一致性。

## 对应代码在哪

- 代码位置：[syncEngine.js](../../../src/services/syncEngine.js)
  符号：`resolveConflict`
  作用：根据用户选择保留本地或采用远端。
- 代码位置：[useSyncStore.js](../../../src/store/useSyncStore.js)
  符号：`addConflict / resolveConflict`
  作用：保存冲突列表并在解决后更新状态。
- 代码位置：[ConflictDialog.jsx](../../../src/components/overlays/ConflictDialog.jsx)
  符号：`ConflictDialog`
  作用：用 Diff 视图把本地和远端版本展示给用户。

## 关键代码怎么读

- 第一步看 [syncEngine.js](../../../src/services/syncEngine.js) 中的 `resolveConflict`，其作用是：根据用户选择保留本地或采用远端。
- 第二步看 [useSyncStore.js](../../../src/store/useSyncStore.js) 中的 `addConflict / resolveConflict`，其作用是：保存冲突列表并在解决后更新状态。
- 第三步看 [ConflictDialog.jsx](../../../src/components/overlays/ConflictDialog.jsx) 中的 `ConflictDialog`，其作用是：用 Diff 视图把本地和远端版本展示给用户。
