# 编辑器主体模块

## 功能是什么

负责源码编辑、所见即所得编辑和多标签管理，是系统的核心交互中心。

## 对应代码在哪

- 代码位置：[EditorContent.jsx](../../../src/components/layout/content/EditorContent.jsx)
  符号：`EditorContent`
  作用：负责把编辑器、预览器和分栏模式编排到同一块内容区域中。
- 代码位置：[useEditorStore.js](../../../src/store/useEditorStore.js)
  符号：`useEditorStore`
  作用：负责标签、活动文档、视图模式和编辑器 UI 状态。
- 代码位置：[editorBuffer.js](../../../src/utils/editorBuffer.js)
  符号：`editorBuffer`
  作用：负责保存高频变化的正文内容，避免每次输入都触发全局重渲染。
- 代码位置：[TabBar.jsx](../../../src/components/layout/tab-bar/TabBar.jsx)
  符号：`TabBar`
  作用：负责多标签的展示、切换、关闭和快捷入口。
- 代码位置：[MonacoEditor.jsx](../../../src/components/editor/MonacoEditor.jsx)
  符号：`MonacoEditor`
  作用：负责源码编辑模式。
- 代码位置：[MilkdownMarkdownEditor.jsx](../../../src/components/editor/MilkdownMarkdownEditor.jsx)
  符号：`MilkdownMarkdownEditor`
  作用：负责所见即所得编辑模式。
- 代码位置：[MarkdownPreview.jsx](../../../src/components/editor/MarkdownPreview.jsx)
  符号：`MarkdownPreview`
  作用：负责只读预览模式。

## 关键代码怎么读

- 第一步看 [EditorContent.jsx](../../../src/components/layout/content/EditorContent.jsx) 中的 `EditorContent`，其作用是：负责把编辑器、预览器和分栏模式编排到同一块内容区域中。
- 第二步看 [useEditorStore.js](../../../src/store/useEditorStore.js) 中的 `useEditorStore`，其作用是：负责标签、活动文档、视图模式和编辑器 UI 状态。
- 第三步看 [editorBuffer.js](../../../src/utils/editorBuffer.js) 中的 `editorBuffer`，其作用是：负责保存高频变化的正文内容，避免每次输入都触发全局重渲染。
- 第四步看 [TabBar.jsx](../../../src/components/layout/tab-bar/TabBar.jsx) 中的 `TabBar`，其作用是：负责多标签的展示、切换、关闭和快捷入口。
- 最后回看其余支撑文件，补齐该功能的边界条件、状态变化和平台差异。
