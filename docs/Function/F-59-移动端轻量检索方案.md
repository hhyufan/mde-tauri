# 移动端轻量检索方案

## 功能是什么

由于 SAF 访问限制，移动端提供轻量化的文件定位方案。

## 对应代码在哪

- 代码位置：[tauriApi.js](../../../src/utils/tauriApi.js)
  符号：`searchFiles`
  作用：在 SAF 场景下把递归全文搜索降级为当前目录名称匹配。
- 代码位置：[SearchModal.jsx](../../../src/components/overlays/SearchModal.jsx)
  符号：`doSearch`
  作用：搜索 UI 统一调用 searchFiles，因此自动继承移动端轻量检索策略。
- 代码位置：[zh_cn.json](../../../src/i18n/locales/zh_cn.json)
  符号：`search hints`
  作用：通过文案区分桌面端递归搜索与移动端轻量搜索提示。

## 关键代码怎么读

- 第一步看 [tauriApi.js](../../../src/utils/tauriApi.js) 中的 `searchFiles`，其作用是：在 SAF 场景下把递归全文搜索降级为当前目录名称匹配。
- 第二步看 [SearchModal.jsx](../../../src/components/overlays/SearchModal.jsx) 中的 `doSearch`，其作用是：搜索 UI 统一调用 searchFiles，因此自动继承移动端轻量检索策略。
- 第三步看 [zh_cn.json](../../../src/i18n/locales/zh_cn.json) 中的 `search hints`，其作用是：通过文案区分桌面端递归搜索与移动端轻量搜索提示。
