# 断网恢复与失败重试

## 功能是什么

网络波动或失败时保留状态并在恢复后继续处理待同步任务。

## 对应代码在哪

- 代码位置：[apiClient.js](../../../src/services/apiClient.js)
  符号：`classifyApiError`
  作用：把异常区分为 offline、server_unreachable 等可恢复类型。
- 代码位置：[syncEngine.js](../../../src/services/syncEngine.js)
  符号：`scheduleRetry / shouldRetry`
  作用：为失败的同步动作安排重试。
- 代码位置：[useSyncStore.js](../../../src/store/useSyncStore.js)
  符号：`failMutation`
  作用：记录 retryCount 和下一次重试时间。

## 关键代码怎么读

- 第一步看 [apiClient.js](../../../src/services/apiClient.js) 中的 `classifyApiError`，其作用是：把异常区分为 offline、server_unreachable 等可恢复类型。
- 第二步看 [syncEngine.js](../../../src/services/syncEngine.js) 中的 `scheduleRetry / shouldRetry`，其作用是：为失败的同步动作安排重试。
- 第三步看 [useSyncStore.js](../../../src/store/useSyncStore.js) 中的 `failMutation`，其作用是：记录 retryCount 和下一次重试时间。
