# 登录

## 功能是什么

用户通过账号进入同步能力体系。

## 对应代码在哪

- 代码位置：[useAuthStore.js](../../../src/store/useAuthStore.js)
  符号：`login / loadToken`
  作用：前端登录动作和会话恢复入口。
- 代码位置：[LoginModal.jsx](../../../src/components/overlays/LoginModal.jsx)
  符号：`handleSubmit`
  作用：登录表单提交入口。
- 代码位置：[auth.controller.ts](../../../mde-server/src/auth/auth.controller.ts)
  符号：`login`
  作用：服务端登录接口。
- 代码位置：[auth.service.ts](../../../mde-server/src/auth/auth.service.ts)
  符号：`validateUser / login`
  作用：用户校验和 JWT 签发。

## 关键代码怎么读

- 第一步看 [useAuthStore.js](../../../src/store/useAuthStore.js) 中的 `login / loadToken`，其作用是：前端登录动作和会话恢复入口。
- 第二步看 [LoginModal.jsx](../../../src/components/overlays/LoginModal.jsx) 中的 `handleSubmit`，其作用是：登录表单提交入口。
- 第三步看 [auth.controller.ts](../../../mde-server/src/auth/auth.controller.ts) 中的 `login`，其作用是：服务端登录接口。
- 第四步看 [auth.service.ts](../../../mde-server/src/auth/auth.service.ts) 中的 `validateUser / login`，其作用是：用户校验和 JWT 签发。
