# 文档同步

## 功能是什么

在多个设备之间同步文档内容。

## 对应代码在哪

- 代码位置：[syncEngine.js](../../../src/services/syncEngine.js)
  符号：`fullSync / queueLocalUpsert / pullRemoteChanges`
  作用：前端文档同步主流程。
- 代码位置：[sync.controller.ts](../../../mde-server/src/sync/sync.controller.ts)
  符号：`getManifest / getChanges / pushFile / pullFile`
  作用：服务端文档同步接口。
- 代码位置：[sync.service.ts](../../../mde-server/src/sync/sync.service.ts)
  符号：`pushFile / getChanges / getFile`
  作用：服务端版本控制和内容读写。

## 关键代码怎么读

- 第一步看 [syncEngine.js](../../../src/services/syncEngine.js) 中的 `fullSync / queueLocalUpsert / pullRemoteChanges`，其作用是：前端文档同步主流程。
- 第二步看 [sync.controller.ts](../../../mde-server/src/sync/sync.controller.ts) 中的 `getManifest / getChanges / pushFile / pullFile`，其作用是：服务端文档同步接口。
- 第三步看 [sync.service.ts](../../../mde-server/src/sync/sync.service.ts) 中的 `pushFile / getChanges / getFile`，其作用是：服务端版本控制和内容读写。
