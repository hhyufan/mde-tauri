# 内容校验与压缩传输

## 功能是什么

同步前计算摘要，对大文本按阈值进行 Gzip 压缩以降低网络开销。

## 对应代码在哪

- 代码位置：[syncEngine.js](../../../src/services/syncEngine.js)
  符号：`COMPRESS_THRESHOLD_BYTES / MAX_REQUEST_BYTES`
  作用：定义压缩阈值和请求上限。
- 代码位置：[syncEngine.js](../../../src/services/syncEngine.js)
  符号：`encodeBody / decodeBody`
  作用：负责压缩、解压和 checksum 计算。
- 代码位置：[push.dto.ts](../../../mde-server/src/sync/dto/push.dto.ts)
  符号：`content / compressed / size`
  作用：服务端接收压缩传输的协议字段。
- 代码位置：[sync-document.schema.ts](../../../mde-server/src/schemas/sync-document.schema.ts)
  符号：`compressed / size / checksum`
  作用：服务端持久化压缩内容和校验信息。

## 关键代码怎么读

- 第一步看 [syncEngine.js](../../../src/services/syncEngine.js) 中的 `COMPRESS_THRESHOLD_BYTES / MAX_REQUEST_BYTES`，其作用是：定义压缩阈值和请求上限。
- 第二步看 [syncEngine.js](../../../src/services/syncEngine.js) 中的 `encodeBody / decodeBody`，其作用是：负责压缩、解压和 checksum 计算。
- 第三步看 [push.dto.ts](../../../mde-server/src/sync/dto/push.dto.ts) 中的 `content / compressed / size`，其作用是：服务端接收压缩传输的协议字段。
- 第四步看 [sync-document.schema.ts](../../../mde-server/src/schemas/sync-document.schema.ts) 中的 `compressed / size / checksum`，其作用是：服务端持久化压缩内容和校验信息。
