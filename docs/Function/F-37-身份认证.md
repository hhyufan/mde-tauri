# 身份认证

## 功能是什么

验证登录信息并决定是否允许继续访问受保护能力。

## 对应代码在哪

- 代码位置：[apiClient.js](../../../src/services/apiClient.js)
  符号：`request/response interceptors`
  作用：负责注入 Bearer Token 和 401 后刷新重试。
- 代码位置：[useAuthStore.js](../../../src/store/useAuthStore.js)
  符号：`refreshToken / logout`
  作用：维护认证状态生命周期。
- 代码位置：[jwt.strategy.ts](../../../mde-server/src/auth/strategies/jwt.strategy.ts)
  符号：`JwtStrategy`
  作用：服务端解析 JWT 并把用户信息挂到请求对象。
- 代码位置：[jwt-auth.guard.ts](../../../mde-server/src/auth/guards/jwt-auth.guard.ts)
  符号：`JwtAuthGuard`
  作用：保护需要认证的接口。

## 关键代码怎么读

- 第一步看 [apiClient.js](../../../src/services/apiClient.js) 中的 `request/response interceptors`，其作用是：负责注入 Bearer Token 和 401 后刷新重试。
- 第二步看 [useAuthStore.js](../../../src/store/useAuthStore.js) 中的 `refreshToken / logout`，其作用是：维护认证状态生命周期。
- 第三步看 [jwt.strategy.ts](../../../mde-server/src/auth/strategies/jwt.strategy.ts) 中的 `JwtStrategy`，其作用是：服务端解析 JWT 并把用户信息挂到请求对象。
- 第四步看 [jwt-auth.guard.ts](../../../mde-server/src/auth/guards/jwt-auth.guard.ts) 中的 `JwtAuthGuard`，其作用是：保护需要认证的接口。
