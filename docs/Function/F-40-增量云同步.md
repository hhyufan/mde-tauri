# 增量云同步

## 功能是什么

只上传发生变化的内容，减少整文件覆盖带来的网络开销。

## 对应代码在哪

- 代码位置：[useSyncStore.js](../../../src/store/useSyncStore.js)
  符号：`cursor state`
  作用：保存增量同步游标。
- 代码位置：[syncEngine.js](../../../src/services/syncEngine.js)
  符号：`pullRemoteChanges / processQueue`
  作用：先处理本地队列，再按 cursor 拉远端变化。
- 代码位置：[sync.controller.ts](../../../mde-server/src/sync/sync.controller.ts)
  符号：`getChanges`
  作用：增量拉取接口。
- 代码位置：[sync.service.ts](../../../mde-server/src/sync/sync.service.ts)
  符号：`getChanges`
  作用：按更新时间返回变更集和新游标。

## 关键代码怎么读

- 第一步看 [useSyncStore.js](../../../src/store/useSyncStore.js) 中的 `cursor state`，其作用是：保存增量同步游标。
- 第二步看 [syncEngine.js](../../../src/services/syncEngine.js) 中的 `pullRemoteChanges / processQueue`，其作用是：先处理本地队列，再按 cursor 拉远端变化。
- 第三步看 [sync.controller.ts](../../../mde-server/src/sync/sync.controller.ts) 中的 `getChanges`，其作用是：增量拉取接口。
- 第四步看 [sync.service.ts](../../../mde-server/src/sync/sync.service.ts) 中的 `getChanges`，其作用是：按更新时间返回变更集和新游标。
