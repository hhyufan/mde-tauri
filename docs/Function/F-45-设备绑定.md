# 设备绑定

## 功能是什么

维护文档在不同设备间的绑定关系与同步身份。

## 对应代码在哪

- 代码位置：[useDeviceStore.js](../../../src/store/useDeviceStore.js)
  符号：`newDeviceId / getId`
  作用：生成并持久化当前设备 ID。
- 代码位置：[syncEngine.js](../../../src/services/syncEngine.js)
  符号：`bindLocalPath / rebindLocalPath`
  作用：把 fileId 和本地路径绑定起来，或在重命名后迁移绑定。
- 代码位置：[sync.service.ts](../../../mde-server/src/sync/sync.service.ts)
  符号：`bindPath`
  作用：服务端写入 deviceBindings 映射。
- 代码位置：[sync-document.schema.ts](../../../mde-server/src/schemas/sync-document.schema.ts)
  符号：`deviceBindings`
  作用：数据库中保存多设备路径关系。

## 关键代码怎么读

- 第一步看 [useDeviceStore.js](../../../src/store/useDeviceStore.js) 中的 `newDeviceId / getId`，其作用是：生成并持久化当前设备 ID。
- 第二步看 [syncEngine.js](../../../src/services/syncEngine.js) 中的 `bindLocalPath / rebindLocalPath`，其作用是：把 fileId 和本地路径绑定起来，或在重命名后迁移绑定。
- 第三步看 [sync.service.ts](../../../mde-server/src/sync/sync.service.ts) 中的 `bindPath`，其作用是：服务端写入 deviceBindings 映射。
- 第四步看 [sync-document.schema.ts](../../../mde-server/src/schemas/sync-document.schema.ts) 中的 `deviceBindings`，其作用是：数据库中保存多设备路径关系。
