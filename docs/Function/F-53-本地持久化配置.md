# 本地持久化配置

## 功能是什么

设置项支持本地持久化，应用重启后能够恢复。

## 对应代码在哪

- 代码位置：[useConfigStore.js](../../../src/store/useConfigStore.js)
  符号：`persist(mde-config)`
  作用：持久化设置。
- 代码位置：[useThemeStore.js](../../../src/store/useThemeStore.js)
  符号：`persist(mde-theme)`
  作用：持久化主题。
- 代码位置：[useEditorStore.js](../../../src/store/useEditorStore.js)
  符号：`persist(mde-editor)`
  作用：持久化侧栏、视图模式等编辑器 UI 状态。
- 代码位置：[useFileStore.js](../../../src/store/useFileStore.js)
  符号：`persist(mde-files)`
  作用：持久化 recent、bookmark、当前目录等文件面板状态。
- 代码位置：[useAuthStore.js](../../../src/store/useAuthStore.js)
  符号：`auth.json via Tauri Store`
  作用：持久化登录态。

## 关键代码怎么读

- 第一步看 [useConfigStore.js](../../../src/store/useConfigStore.js) 中的 `persist(mde-config)`，其作用是：持久化设置。
- 第二步看 [useThemeStore.js](../../../src/store/useThemeStore.js) 中的 `persist(mde-theme)`，其作用是：持久化主题。
- 第三步看 [useEditorStore.js](../../../src/store/useEditorStore.js) 中的 `persist(mde-editor)`，其作用是：持久化侧栏、视图模式等编辑器 UI 状态。
- 第四步看 [useFileStore.js](../../../src/store/useFileStore.js) 中的 `persist(mde-files)`，其作用是：持久化 recent、bookmark、当前目录等文件面板状态。
- 最后回看其余支撑文件，补齐该功能的边界条件、状态变化和平台差异。
