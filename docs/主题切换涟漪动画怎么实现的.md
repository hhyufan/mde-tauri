# 主题切换涟漪动画怎么实现的

## 1. 这篇文档讲什么

这篇文档专门解释当前项目里的“主题切换涟漪动画”是怎么实现的。

这里说的“涟漪”并不是单独画一个按钮点击波纹组件，而是更接近 Google 常说的那种页面主题切换范式：

- 基于 `View Transition API`
- 用圆形扩散的 `clip-path` 做 `Circular Reveal`

在当前项目里，主题切换动画的核心实现只有一套，入口就在侧边栏的主题按钮上。

---

## 2. 它本质上用的是什么技术

当前项目的主题切换动画并不是纯 CSS keyframes，也不是单独做一个遮罩层，而是：

1. 点击主题按钮
2. 调用 `document.startViewTransition()`
3. 在切换主题后，对 `::view-transition-new(root)` 执行一段 `clipPath: circle(...)` 动画
4. 让新主题从按钮中心向外扩散覆盖整个页面

最核心的实现位于：

- 动画入口：[Sidebar.jsx:L400-L415](../src/components/layout/sidebar/Sidebar.jsx#L400-L415)
- 主题状态切换：[useThemeStore.js:L24-L42](../src/store/useThemeStore.js#L24-L42)
- 关闭默认过渡动画：[index.scss:L116-L119](../src/assets/styles/index.scss#L116-L119)

## 4. `handleThemeSwitch()` 是怎么做涟漪扩散的

代码位置：[Sidebar.jsx:L400-L415](../src/components/layout/sidebar/Sidebar.jsx#L400-L415)

```js
function handleThemeSwitch() {
  const element = themeButtonRef.current;
  if (!element) { toggleTheme(); return; }
  const rect = element.getBoundingClientRect();
  const x = rect.left + rect.width / 2;
  const y = rect.top + rect.height / 2;
  if (!document.startViewTransition) { toggleTheme(); return; }
  const transition = document.startViewTransition(() => { toggleTheme(); });
  transition.ready.then(() => {
    const radius = Math.sqrt(Math.max(x, window.innerWidth - x) ** 2 + Math.max(y, window.innerHeight - y) ** 2);
    document.documentElement.animate(
      { clipPath: [`circle(0 at ${x}px ${y}px)`, `circle(${radius}px at ${x}px ${y}px)`] },
      { duration: 300, pseudoElement: '::view-transition-new(root)' }
    );
  });
}
```

这段代码可以拆成 5 个步骤来看。

### 4.1 先拿到按钮中心点

```js
const rect = element.getBoundingClientRect();
const x = rect.left + rect.width / 2;
const y = rect.top + rect.height / 2;
```

这一步是为了确定圆形扩散动画的起点，让主题切换看起来像是从按钮位置“涌出来”的。

### 4.2 不支持时优雅降级

```js
if (!document.startViewTransition) { toggleTheme(); return; }
```

这说明项目并没有强依赖 View Transition API。如果浏览器或 WebView 不支持，就直接切换主题，不做动画。

### 4.3 在视图过渡上下文里切换主题

```js
const transition = document.startViewTransition(() => { toggleTheme(); });
```

这里不是先切主题再动画，而是把 `toggleTheme()` 放进 `startViewTransition()` 的回调中。这样浏览器才会生成 `old(root)` / `new(root)` 这两层过渡快照。

### 4.4 计算覆盖全屏所需半径

```js
const radius = Math.sqrt(
  Math.max(x, window.innerWidth - x) ** 2 +
  Math.max(y, window.innerHeight - y) ** 2
);
```

这个半径不是随便写死的，而是动态算出从按钮中心到视口最远角落的距离。只有这样，圆形扩散最终才能完整覆盖整个页面。

### 4.5 对新主题层做圆形揭示动画

```js
document.documentElement.animate(
  { clipPath: [`circle(0 at ${x}px ${y}px)`, `circle(${radius}px at ${x}px ${y}px)`] },
  { duration: 300, pseudoElement: '::view-transition-new(root)' }
);
```

这里的核心点有两个：

- 使用 `clipPath: circle(...)` 从 `0` 扩到 `radius`
- 目标不是普通 DOM，而是 `::view-transition-new(root)`

也就是说，真正被“揭开”的是已经切换好主题的新页面快照。

## 5. 真正的主题切换并不是在 `Sidebar` 里完成的

`Sidebar` 只负责“何时切换”和“怎么做动画”，真正切换主题状态的是 `useThemeStore`。

代码位置：[useThemeStore.js:L24-L42](../src/store/useThemeStore.js#L24-L42)

```js
setTheme: (theme, meta = {}) => {
  document.documentElement.dataset.theme = theme;
  set({
    theme,
    themeUpdatedAt: meta.updatedAt ?? Date.now(),
  });
},

toggleTheme: () => {
  const next = get().theme === 'dark' ? 'light' : 'dark';
  document.documentElement.dataset.theme = next;
  set({
    theme: next,
    themeUpdatedAt: Date.now(),
  });
},
```

这里可以看出：

- 真正生效的主题标识写在 `document.documentElement.dataset.theme`
- Zustand store 同步保存当前主题状态

因此涟漪动画和主题状态切换其实是两个层面：

- `Sidebar` 处理过渡表现
- `useThemeStore` 处理主题状态

## 6. 为什么还要在全局样式里关闭默认过渡动画

项目里并没有直接使用浏览器默认的 View Transition 动画，而是用手写的圆形扩散动画来接管。

相关代码位置：[index.scss:L116-L119](../src/assets/styles/index.scss#L116-L119)

这一段的作用是：

- 禁掉 `::view-transition-old(root)` 和 `::view-transition-new(root)` 的默认动画
- 避免默认过渡和自定义 `clip-path` 动画叠加

这一步很重要，否则浏览器自带的淡入淡出会和手写涟漪效果打架，视觉上会变得很乱。

## 8. 为什么说这是 Google 那种“切换范式”

你前面提到的“Google 说的一种切换泛式”，放到当前实现里，对应的就是：

- 技术能力：`View Transition API`
- 动画形态：`Circular Reveal`
- 设计语境：接近 `Material Motion`

本项目虽然没有引入完整的 Material Design 动画体系，但主题切换这一段的思路就是典型的“以交互点为中心，对新状态做径向揭示”。

所以如果你后面写论文或技术说明，可以用下面这组说法：

- 使用 `View Transition API` 实现主题切换过渡
- 采用 `Circular Reveal` 风格的圆形扩散动画
- 通过 `clip-path` 与伪元素 `::view-transition-new(root)` 完成新主题揭示

## 9. 这套实现为什么设计得比较合理

从代码结构上看，这套实现有 4 个优点。

### 9.1 有降级策略

如果运行环境不支持 `startViewTransition`，就直接 `toggleTheme()`，不会因为动画能力缺失而影响主题切换本身。

### 9.2 动画和状态切换分层清楚

- `Sidebar` 负责过渡动画
- `useThemeStore` 负责主题状态

这使得后面如果要换一种主题动画，只需要改 `handleThemeSwitch()`，不必动主题状态层。

### 9.3 扩散中心来自真实点击位置

动画不是固定从页面中心展开，而是从主题按钮中心展开，交互感更强。

### 9.4 下游组件统一响应 `data-theme`

只要主题 store 改变了 `document.documentElement.dataset.theme`，Monaco、Milkdown、预览、Mermaid 等模块都能根据同一个主题标识进行更新。

## 11. 结论

当前项目的主题切换涟漪动画，本质上是：

- 用 `startViewTransition()` 建立过渡上下文
- 用 `toggleTheme()` 切换 `data-theme`
- 用 `clip-path: circle(...)` 对 `::view-transition-new(root)` 做圆形扩散揭示

所以它不是普通按钮波纹，而是一种页面级的主题过渡动画。你如果要给它一个更准确的技术名称，可以说：

- “基于 `View Transition API` 的 `Circular Reveal` 主题切换动画”

这个表述最贴近当前项目代码的真实实现。
