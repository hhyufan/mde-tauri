# 论文代码展示端 Spec

## Why
当前仓库已经有论文 Markdown、按功能拆分的功能文档，以及完整的源码实现，但这些内容仍然分散在多个目录中。对于想从论文直接理解“某个功能在代码里是如何实现”的读者，缺少一个独立的、以论文为起点的可视化展示端，会显著抬高讲解、演示和答辩成本。

## What Changes
- 新增一个独立的 React 展示端，使用 Ant Design 作为 UI 基础，并延续 MDE 的整体视觉风格
- 展示端默认加载论文 Markdown 作为首页正文，而不是代码编辑器界面
- 论文中与 `F-xx` 功能编号相关的脚注或标记需被识别为可点击入口
- 点击 `F-xx` 入口后，展示端需弹出功能说明窗口，展示相对于仓库根目录的代码树、相关文件、关键代码片段和讲解信息
- 代码展示需支持“按文件查看”和“按片段查看”，帮助读者理解功能与源码之间的映射关系
- 展示端需尽量复用当前已经生成的“按功能拆分文档”，并补充结构化数据，避免手工重复维护两套说明
- 该展示端必须作为独立入口存在，不影响当前主应用编辑、同步和文件管理流程

## Impact
- Affected specs: 论文演示体验、源码讲解能力、答辩展示能力、新读者理解路径
- Affected code: 前端独立展示入口、论文 Markdown 渲染层、功能编号映射数据、代码片段抽取逻辑、弹窗与代码树 UI、功能说明文档产物

## ADDED Requirements
### Requirement: 提供独立的论文代码展示端
系统 SHALL 提供一个独立的 React 展示端，用于围绕论文内容展示功能与源码的对应关系，而不是复用现有编辑器主界面直接改造。

#### Scenario: 用户打开展示端
- **WHEN** 用户进入该展示端入口
- **THEN** 页面应直接展示论文 Markdown 正文，并采用与 MDE 风格接近的阅读型布局

#### Scenario: 展示端与主应用隔离
- **WHEN** 开发者实现该展示端
- **THEN** 该入口应保持独立，不得破坏主编辑器、同步、文件管理等既有流程

### Requirement: 将论文中的功能编号转成交互式脚注入口
系统 SHALL 识别论文中出现的 `F-xx` 功能编号或映射标记，并将其渲染为可点击的交互式脚注入口。

#### Scenario: 读者阅读论文时看到功能编号
- **WHEN** 论文正文中出现 `F-10`、`F-40` 一类功能编号
- **THEN** 这些编号应以明显但不打断阅读的方式展示，并允许点击

#### Scenario: 读者点击功能脚注
- **WHEN** 用户点击某个功能编号
- **THEN** 系统应打开功能详情弹窗，而不是跳离当前阅读上下文

### Requirement: 在弹窗中展示功能与源码的映射关系
系统 SHALL 在功能详情弹窗中展示该功能对应的代码树、相关文件和关键代码片段，并说明这些代码分别承担什么职责。

#### Scenario: 查看功能详情
- **WHEN** 用户打开某个 `F-xx` 功能的详情
- **THEN** 弹窗中应至少包含功能简介、相关文件列表、相对根目录路径、关键符号名称和代码片段说明

#### Scenario: 查看相对根目录的代码树
- **WHEN** 用户阅读功能详情
- **THEN** 系统应展示该功能涉及的代码树或文件层级，并突出本次功能真正相关的文件

#### Scenario: 查看关键代码片段
- **WHEN** 用户切换到某个相关文件
- **THEN** 系统应展示该文件的关键代码片段，并附带“为什么这一段重要”的解释

### Requirement: 以功能文档作为展示数据基础
系统 SHALL 优先复用当前仓库中已经整理出的按功能拆分文档，并在必要时补充结构化映射数据，确保论文、功能和源码之间的关系可维护。

#### Scenario: 新增或调整某个功能映射
- **WHEN** 开发者补充 `F-xx` 对应的文件、片段或说明
- **THEN** 展示端应能从统一的数据源读取这些信息，而不是把说明硬编码在 UI 组件中

#### Scenario: 维护论文与代码的一致性
- **WHEN** 某个功能文档更新了代码位置或关键说明
- **THEN** 展示端应能同步读取更新后的映射结果，降低重复维护成本

### Requirement: 保持阅读体验和展示体验兼顾
系统 SHALL 在阅读论文时保持正文连续性，同时让用户快速切入功能讲解，不把展示端做成纯代码浏览器。

#### Scenario: 用户以论文阅读为主
- **WHEN** 用户尚未点击任何功能脚注
- **THEN** 页面应优先表现为高可读性的论文阅读界面

#### Scenario: 用户开始追源码
- **WHEN** 用户点击某个功能脚注并展开代码详情
- **THEN** 系统应提供清晰的文件切换、片段展示和说明区域，帮助用户理解该功能的实现主线

## MODIFIED Requirements
### Requirement: 功能文档的使用方式
现有按功能拆分的功能文档在新增展示端后，不再仅作为静态阅读材料存在，还必须能够被展示端复用为“功能说明层”的一部分，用于支撑弹窗内的功能简介、代码位置和关键代码阅读顺序。

## REMOVED Requirements
### Requirement: Old Feature
**Reason**: 本次变更不涉及移除现有功能
**Migration**: 无
